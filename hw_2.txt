 1. Создать файл “new.json” - touch new.json
 2. Добавить файл под гит. - git add touch new.json
 3. Закоммитить файл. - git commit -m "file new.json"
 4. Отправить файл на внешний GitHub репозиторий. - git push
 5. Создать файлы new2.json, new2.xml, new2.txt - touch new2.json new2.xml new2.txt
 6. Добавить файлы под гит. - git add new2.json new2.xml new2.txt
 7. Закоммитить файлы. - git commit -m "message"
 8. Отправить файлы на внешний GitHub репозиторий. - git push
 9. Отредактировать содержание файла “new2.txt” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). - vim new2.txt (записать данные)
 10. Отредактировать содержание файла “new2.json” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате JSON. - vim new2.json
 11. Добавить и закоммитить “new2.txt” одной строчкой. - git commit -a -m "file new2.txt"
 12. Добавить и закоммитить “new2.json” одной строчкой. - git commit -a -m "new2.json"
 13. Отправить изменения на внешний репозиторий. - git push
 14. Отредактировать содержание файла “new2.xml” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате XML.
(<information>
<name>Ivan</name>
<middle_name>Ivanovych</middle_name>
<last name>Ivanov</last_name>
<age>30</age>
<pets>2</pets>
<future_salary>3000</future_salary>
</information>)

 15. Добавить и закоммитить “new2.xml” одной строчкой. - git commit -a-m "file new2.xml"
 16. Отправить изменения на внешний репозиторий. - git push
 17. Отправить предыдущее домашнее задание по GitBash Terminal на внешний репозиторий. - git add hw.txt; git commit -m "hw_1.txt"; git push
 18. В веб интерфейсе GitHub создать файл new3.txt. - Зайти в созданный репозиторий, создать файл new3.txt
 19. Отредактировать в веб интерфейсе содержание файла “new3.txt” - написать информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить).
 20. Сделать Commit changes (сохранить) изменения на веб интерфейсе. - git pull
